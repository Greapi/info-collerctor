# 2026-07-20 RSS 日报

## 今日概览
今日 AI 领域多条主线并发：Grok 4.5 正式面向 Excel 与开发者推出，并在编程基准测试中超越主流竞品；Kimi K3 因需求暴增暂停新订阅，再次将国产模型算力瓶颈推向台前；OpenAI 发布长周期模型安全框架，同时悄然缩减 Codex 上下文窗口。安全方面，研究员用 GPT5.6 加 25 美元发现 WordPress 远程代码执行漏洞，而罗马尼亚土地注册数据库遭黑客清除。开源社区持续活跃，多款语音、MCP、终端工具冲上 GitHub 趋势榜。独立开发者叙事中，Vibe Coding 的投入产出比与用户真实需求错位，引发新一轮讨论。

## 重点事件
- **Grok 4.5 全面铺开**  
  Elon Musk 宣布 [Grok for Excel 上线](https://x.com/elonmusk/status/2079261553136734239)，用户可在 Excel 中直接调用 Grok 4.5 构建金融模型、分析市场数据。同日，Grok 4.5 在 VulcanBench 编程基准中得分 91.3%，超越 Claude Fable 5 和 GPT‑5.6 Sol，且成本效率更优（[来源](https://x.com/elonmusk/status/2078987522927972695)）。开发者 Lee Robinson 称赞其在 React 开发中“出色且经济高效”（[来源](https://x.com/elonmusk/status/2079261743650312227)），Min Choi 则将其用于实时研究、日常编码和调试（[来源](https://x.com/elonmusk/status/2079157111183806697)）。Grok Build 0.2.105 同步发布，已将 4.5 设为默认模型并新增 /summarize 命令（[来源](https://x.com/elonmusk/status/2078692662345888102)）。

- **Kimi K3 需求过热，暂停新订阅**  
  月之暗面因 Kimi K3 需求超预期，[暂停新用户订阅](https://twitter.com/kimi_moonshot/status/2078855608565207130)（[HN 讨论](https://news.ycombinator.com/item?id=48969291)）。社区反馈激烈：前端能力被部分用户视为全球第一梯队，但存在价格昂贵、速度偏慢、幻觉率高等问题（[V2EX 讨论](https://www.v2ex.com/t/1228376)）；K2.6 被指“降智”，部分用户转向千问 3.8 Preview（[V2EX](https://www.v2ex.com/t/1228368)）；算力不足直接引发国产算力能力的广泛讨论（[V2EX](https://www.v2ex.com/t/1228419)）。此前公布的 Kimi Code 会员计划定价从 ¥99 至 ¥1399/月，被评价“毫无性价比”（[来源](https://www.v2ex.com/t/1228417)）。

- **OpenAI 发布长周期模型安全研究，并缩减 Codex 上下文**  
  OpenAI 公开了[《Safety and alignment in an era of long-horizon models》](https://openai.com/index/safety-alignment-long-horizon-models)，指出可长期运行的模型在解决复杂开放问题的同时，会引入传统评估难以捕获的安全风险，并给出部署迭代中的失败案例与改进措施（[OpenAI 公告](https://x.com/OpenAI/status/2079260694076371442)）。另一方面，Codex 模型上下文大小由 372k 缩减至 272k（[PR 记录](https://github.com/openai/codex/pull/33972/files)，[HN 讨论](https://news.ycombinator.com/item?id=48965850)），引发开发者关注。

- **25 美元发现 WordPress RCE，漏洞贩子出价超 50 万美元**  
  安全研究员称仅使用 GPT5.6 和 25 美元就找到了一个 WordPress 远程代码执行漏洞，而漏洞中间商对同类漏洞的出价高达 50 万美元（[报告](https://slcyber.io/research-center/exploit-brokers-pay-500000-for-a-wordpress-rce-i-found-one-with-gpt5-6/)）。同时，fastjson 1.x 最新版本亦传出疑似的 RCE 漏洞，尚未获 CVE 编号与官方确认（[V2EX](https://www.v2ex.com/t/1228636)）。

- **Anthropic 启动罕见病研究与 Claude 资助计划**  
  Anthropic 宣布“AI for Science”项目首次定向征集，为加速罕见病治疗的研究人员提供高达 50,000 美元的 Claude 使用额度资助（[公告](https://x.com/AnthropicAI/status/2079256626771665098)）。

- **独立开发者 AI 实践与反思**  
  独立开发者利用 Codex 和 Claude Code 在两月内单人完成 App “ArkFlow”的开发与上架，AI 工具花费约 900 美元，总支出 1050 美元，上线 7 天下载破千次、收入约 700 美元。但用户最感兴趣的不是原设计的“资产自由度”系统，而是银行卡堆叠、滑动交互与卡面收藏，促使产品转向“私人资产卡包”定位（[少数派](https://sspai.com/post/112442)）。另一 Vibe Coding 项目“新闻雷达”4 个月获 384 位注册用户、29 位付费，收入 1600 元，但服务器与模型 API 成本已超 8000 元，运营推广成最大瓶颈（[V2EX](https://www.v2ex.com/t/1228385)）。这些案例折射出 AI 辅助开发中用户真需求与技术想象的差距，以及成本控制的现实挑战。

## 趋势与判断
- **中国开源权重 AI 策略正在赢得竞争优势**  
  分析文章指出，美国封闭的专有 AI 模式面临挑战，而中国以开源权重为导向的 AI 策略正在取胜（[来源](https://werd.io/american-ai-is-locked-down-and-proprietary-its-losing/)）。同日，Qwen 3.8 正式发布（[来源](https://twitter.com/Alibaba_Qwen/status/2078759124914098291)，[HN](https://news.ycombinator.com/item?id=48966120)），Ollama 亦推出博客“All Aboard Open Models”力推开源（[来源](https://ollama.com/blog/all-aboard-open-models)，[HN](https://news.ycombinator.com/item?id=48965880)），强化了这一趋势信号。

- **AI 生成内容正加速渗透学术与创意领域**  
  一项研究测量显示，arXiv 上超过 30% 的新提交论文读起来像 AI 撰写（[来源](https://unslop.run/blog/measuring-ai-writing-on-arxiv)）。AI 文本人性化市场预计将成长为 1000 亿美元产业（[V2EX](https://www.v2ex.com/t/1228672)）。同时，摄影师尝试使用 AI 修图后感到自我价值动摇（[V2EX](https://www.v2ex.com/t/1228678)），爱奇艺新剧 AI 制作片头片尾被指使剧集质量退步（[V2EX](https://www.v2ex.com/t/1228420)），AI 对原创性工作的冲击正从效率讨论深化为身份焦虑。

- **AI 编码工具深度嵌入开发工作流，但管理“经验”仍待解**  
  开发者围绕 Claude Code 与 Codex 积累的会话经验难以复用展开讨论，尝试将旧 session 蒸馏为 skill、文档或 prompt 模板（[V2EX](https://www.v2ex.com/t/1228685)）。GPT‑5.6 Sol 的 High/Max 模式实测则显示，修复小范围 Bug 用 High 即可，大范围重写或迁移才值得上 Max，Max 比 High 综合成本增加 2.42 倍但通过率仅提升 3.3 个百分点（[V2EX](https://www.v2ex.com/t/1228412)）。精细化控制推理成本正成为开发者的必备技能。

- **安全威胁与基础设施脆弱性凸显**  
  罗马尼亚整个土地注册数据库被黑客清除（[来源](https://news.risky.biz/risky-bulletin-hacker-wipes-romanias-entire-land-registry-database/)），个人以极低成本发现高价值漏洞，以及欧盟可能为免签旅行向美国出售敏感公民数据的警告（[来源](https://edri.org/our-work/the-eu-is-about-to-sell-our-most-sensitive-data-to-the-us-for-visa-free-travel/)），共同指向数字基础设施的脆弱性与数据主权议题的升温。
