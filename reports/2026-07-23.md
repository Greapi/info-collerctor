## 今日概览
今日科技领域呈现多线爆发。xAI 正式上线 Grok 4.5 并发布 Grok Build 重磅更新，Tesla 用超 120 亿英里数据证明 FSD 安全性。梁文锋在投资者交流会中阐释以善意驱动的 AGI 路线图。工业边缘网关首次接入 MCP，AI 开始触碰物理世界；同时 OpenAI 上线健康功能，但也因意外攻击 Hugging Face 陷入争议。AI 编程工具的付费透明度、缓存机制等开发者体验议题持续发酵，欧盟 Chat Control 1.0 的落地敲响隐私警钟。

## 重点事件

- **xAI 全平台上线 Grok 4.5，工具链大幅增强**  
  Grok 4.5 已在 X、iOS、Android 及网页端全面可用，以极高 Token 效率和速度著称，能够以极低成本完成与 Fable 同等质量的任务 [Grok 4.5 now on all platforms](https://x.com/elonmusk/status/2080351573675544984)。  
  在科研侧，运行在 Grok 4.5 Medium 上的 “Capy” 在 Slack 中用 8 分钟就找到了开放约 30 年的图论猜想（Graffiti Conjecture 284）的反驳 [Grok 4.5 just solved a graph theory conjecture that has been open for ~30 years](https://x.com/elonmusk/status/2080165738464280725)。  
  Grok Build 发布 v0.2.111，增强开发者控制、账户可见性、诊断及会话可靠性；配合 Browser Use CLI 时在浏览器任务上宣称超越 Codex 和 Claude Code [Grok Build gets better every day](https://x.com/elonmusk/status/2080170228554903796)。  
  新增功能包括：文档新增权限、计划模式、沙箱、子代理页面 [RT Tech Dev Notes](https://x.com/elonmusk/status/2080172169846485231)；上线 Exa 语义搜索插件 [RT X Freeze: Exa plugin](https://x.com/elonmusk/status/2080061639932665883)；以及 Workflows 固定结构多代理流水线 [Try Workflows](https://x.com/elonmusk/status/2079842345668022641)。

- **Tesla FSD 安全里程突破 120 亿英里，Robotaxi 零事故**  
  Tesla 表示，FSD (Supervised) 基于超过 120 亿英里行驶数据，碰撞间隔里程是手动驾驶的 2 倍，能预测闯红灯、车辆突然转向等危险场景 [RT Ashok Elluswamy](https://x.com/elonmusk/status/2080171935431008305)。马斯克指出 Robotaxi 扩展必须极度谨慎，并提到美国每年约 3 万至 4 万起汽车相关死亡，延缓技术就是伤害公民 [Yeah](https://x.com/elonmusk/status/2080350615658749956)。最新数据显示，Robotaxi 已行驶超 38 万英里，零显著事故 [RT Tesla: 0 notable incidents](https://x.com/elonmusk/status/2080171886500335632)。

## 趋势与判断

1. **大模型部署成本加速下降，开源体系冲击闭源定价**  
   Grok 4.5 以极低成本实现接近 Fable 的质量，GPT-5.6 限时降至 0.02 倍率 [GPT-5.6 限时优惠](https://www.v2ex.com/t/1229153#reply4)，梁文锋明确六倍利润线即行业合理价格。结合 API 拼车单日消耗超 30 亿 Token [OpenAI API 拼车](https://www.v2ex.com/t/1229415#reply1)，Token 成本正从痛点变为水电式基础设施，深度影响 AI 应用生态。

2. **Agent 工作流走向结构化与物理世界**  
   Grok Build 推出 Workflows 固定多代理流水线，EdgeX 的 MCP 支持则让 AI 可直接指挥工业设备。从软件 Agent 到硬件 Agent 的跨越在今年加速，如 Screenpipe 通过 24/7 屏幕录制为代理提供上下文 [Launch HN: Screenpipe](https://news.ycombinator.com/item?id=49024620)，AgentDock 实现多设备控制 [AgentDock](https://www.v2ex.com/t/1229409#reply0)。

3. **AI 编程工具体验上升为关键竞争维度**  
   Kimi Code 因付费不透明遭遇投诉，Claude Code 缓存失效陷阱引发大规模讨论，“规划用 Claude，代码用 GPT”成为社区实践 [编程能力排名](https://www.v2ex.com/t/1229132#reply12)。开发者对模型能力之外的限额透明、缓存效率、工作流集成愈发敏感，工具链的“爽感”直接影响用户粘性。

4. **隐私与安全成为 AI 时代的新裂缝**  
   欧盟 Chat Control 1.0 以儿童保护之名允许扫描私人通讯，加上陕西电信 DNS 劫持用户域名 [陕西电信 DNS 劫持](https://www.v2ex.com/t/1229390#reply5)、LG 智能电视内置住宅代理 SDK 被禁 [LG 封禁含住宅代理 SDK 应用](https://www.v2ex.com/t/1229156#reply1)，物理世界与数字世界的监控边界正在模糊，隐私保护技术（如 E2EE）面临更大挑战。
