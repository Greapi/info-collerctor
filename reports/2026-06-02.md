## 趋势与判断
- **AI 落地瓶颈在业务翻译，而非 API**：OpenAI 和 Anthropic 开始向客户派驻 FDE（前沿部署工程师），凸显“最后一公里”的工程化难题。建议企业在使用厂商 FDE 时搭配中立架构评审，防范供应商锁定。([职场话题：AI 越来越难落地](https://www.v2ex.com/t/1217425#reply1))
- **AI 工程师角色急速分化**：Eval 工程师、Harness 工程师、LLMOps、AI 数据工程师等新岗位涌现，企业正围绕可评估、可约束、可运维的 AI 工程化体系构建团队。([同上](https://www.v2ex.com/t/1217425#reply1))
- **搜索即代码**：Perplexity 提出将搜索重新思考为代码生成，可能改变下一代信息检索范式。([Rethinking Search as Code Generation](https://research.perplexity.ai/articles/rethinking-search-as-code-generation))
- **企业级 AI Coding 架构走向三层分离**：Harness Coding 实践中，“人类需求层－工程契约层－代码执行层”搭配规划、实现、评估 Agent 流水线，成为保障代码质量与可控性的主流架构。([Harness 架构](https://www.v2ex.com/t/1217155#reply0))
- **隐私与监控边界再讨论**：Larry Ellison 的“全民记录”言论和马斯克转发的 Meta AI 安全风险，共同折射出 AI 时代隐私侵蚀的深层焦虑。
- **免费额度缩水成为用户痛点**：OpenAI Codex 转为月重置、GitHub Copilot 积分受限，可能推动用户转向自建或社区中转方案。
- **开源实时数字人与记忆增强代理**：OpenTalking、MemoryProxy 等项目活跃，表明实时交互、长期记忆已成为 LLM 应用的关键补全能力。
